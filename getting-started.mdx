# Getting Started with Fleet AI

Welcome to Binalyze Fleet AI! This guide will help you quickly start using our AI-powered cybersecurity platform to enhance your digital forensics and incident response operations.

## Accessing Fleet AI

### **Platform Access**
Fleet AI is available through our web platform at [ai.binalyze.com](https://ai.binalyze.com). Access is provided through enterprise licensing with your organization.

### **Authentication**
- **Enterprise SSO**: Use your organization's single sign-on credentials
- **JWT Authentication**: Secure token-based authentication for seamless integration
- **Session Management**: Persistent sessions across browser sessions for continuity

<Note>
Contact your system administrator or [support@binalyze.com](mailto:support@binalyze.com) if you need access credentials or have authentication issues.
</Note>

## Your First Chat

### **Starting a Conversation**
1. **Login** to the Fleet AI platform using your credentials
2. **Choose Your Interface**: Select from available chat interfaces
3. **Begin Typing**: Start with a cybersecurity question or describe your task

### **Example First Interactions**
```
"I need help analyzing a suspicious executable file"
"Generate a YARA rule for detecting ransomware behavior"
"Create a Sigma rule for detecting lateral movement"
"Help me investigate a potential data exfiltration incident"
```

### **Understanding Agent Responses**
- **Streaming Responses**: Real-time text generation for immediate feedback
- **Tool Execution**: Visual indicators when agents use specialized tools
- **Context Awareness**: Agents remember conversation history for follow-up questions
- **Expert Routing**: Automatic redirection to specialized agents when needed

## Navigation Basics

### **Main Chat Interface**
- **Blacklight Team**: Primary chat interface for general cybersecurity assistance
- **Message History**: Persistent conversation history across sessions
- **Message Editing**: Ability to edit and regenerate responses
- **Context Switching**: Seamless transitions between different investigation topics

### **Expert Agent Access**
Use expert mentions to access specialized agents directly:
- **@yara**: Direct access to YARA Agent for malware detection rules
- **@sigma**: Direct access to Sigma Agent for log analysis rules  
- **@osquery**: Direct access to OSQuery Agent for endpoint queries
- **@fleet**: Access to general Fleet Agent for DFIR guidance

### **Team Coordination**
- **Detection Team**: Coordinated workflows involving multiple detection agents
- **Intelligent Routing**: Automatic selection of the most appropriate agent
- **Multi-Agent Workflows**: Complex investigations involving multiple specialists

## Common Workflows

### **Incident Response**
1. **Describe the Incident**: Provide available details about the security event
2. **Get Analysis Guidance**: Receive expert recommendations for investigation steps
3. **Generate Detection Rules**: Create rules to detect similar threats
4. **Document Findings**: Use Fleet AI to help document investigation results

### **Malware Analysis**
1. **Upload Indicators**: Provide file hashes, signatures, or behavioral descriptions
2. **Expert Analysis**: Get detailed malware family identification and analysis
3. **Rule Generation**: Receive YARA rules for detecting the malware
4. **Deployment Guidance**: Learn how to implement detection in your environment

### **Threat Hunting**
1. **Define Hunting Hypothesis**: Describe what threats you're searching for
2. **Get Hunting Queries**: Receive Sigma and OSQuery rules for threat detection
3. **Iterative Refinement**: Refine queries based on initial results
4. **Proactive Detection**: Deploy rules for ongoing threat monitoring

## Best Practices

### **Effective Communication**
- **Be Specific**: Provide detailed context about your cybersecurity challenge
- **Include Examples**: Share file hashes, log samples, or IOCs when available
- **Ask Follow-ups**: Don't hesitate to request clarification or additional analysis
- **Iterate**: Refine your questions based on initial responses

### **Security Considerations**
- **Sensitive Data**: Avoid sharing actual passwords or personal information
- **Classification**: Consider data classification when sharing context
- **Validation**: Always validate generated rules in test environments first
- **Documentation**: Maintain records of AI-generated analysis for auditing
